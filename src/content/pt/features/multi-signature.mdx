## Visão Geral

A tecnologia de multi-assinatura (multi-sig) adiciona uma camada extra de segurança ao seu TBC Wallet, exigindo múltiplas aprovações antes que qualquer transação possa ser executada. Este recurso é essencial para proteger ativos de alto valor e é ideal para uso pessoal e carteiras compartilhadas.

## O que é Multi-Assinatura?

Multi-assinatura é um recurso de segurança que requer múltiplas chaves privadas para autorizar uma transação. Em vez de uma única chave controlando sua carteira, carteiras multi-sig requerem um número predeterminado de assinaturas (por exemplo, 2 de 3, 3 de 5) para aprovar qualquer transação.

### Como Funciona

1. **Configuração**: Configure sua carteira multi-sig com múltiplos detentores de chaves
2. **Solicitação de Transação**: Quando uma transação é iniciada, requer aprovação
3. **Processo de Aprovação**: Os signatários designados devem aprovar a transação
4. **Execução**: Uma vez que o número necessário de aprovações é atingido, a transação é executada

## Benefícios de Segurança

### Proteção Contra Ponto Único de Falha

- **Perda de Chave**: Se uma chave for perdida, seus fundos permanecem seguros
- **Prevenção de Roubo**: Uma única chave comprometida não pode drenar sua carteira
- **Ameaças Internas**: Reduz o risco de insiders maliciosos

### Controle Aprimorado

- **Carteiras Compartilhadas**: Perfeito para empresas ou famílias gerenciando fundos compartilhados
- **Governança**: Permite tomada de decisão democrática para gestão de fundos
- **Rastreamento de Auditoria**: Todas as aprovações são registradas para transparência

### Conformidade Regulatória

- **Uso Corporativo**: Atende aos requisitos para gestão de tesouraria corporativa
- **Conformidade**: Ajuda a atender requisitos regulatórios para custódia de fundos
- **Responsabilidade**: Cadeia de aprovação clara para auditorias financeiras

## Guia de Configuração

### Pré-requisitos

Antes de configurar multi-assinatura, certifique-se de ter:

- Uma conta TBC Wallet
- Múltiplos dispositivos confiáveis ou detentores de chaves
- Compreensão do processo de aprovação

### Configuração Passo a Passo

1. **Acesse Configurações Multi-Sig**: Navegue até as configurações de Segurança no TBC Wallet
2. **Escolha Configuração**: Selecione sua configuração multi-sig (por exemplo, 2-de-3, 3-de-5)
3. **Adicione Detentores de Chaves**: Adicione os endereços ou dispositivos que manterão as chaves de assinatura
4. **Configure Limiar**: Defina o número mínimo de aprovações necessárias
5. **Teste Transação**: Execute uma transação de teste para verificar a configuração
6. **Ative**: Habilite a proteção multi-assinatura para sua carteira

### Opções de Configuração

#### Configuração 2-de-3

- Requer 2 de 3 assinaturas
- Ideal para uso pessoal com chaves de backup
- Equilíbrio entre segurança e conveniência

#### Configuração 3-de-5

- Requer 3 de 5 assinaturas
- Melhor para organizações ou carteiras compartilhadas
- Maior segurança com mais flexibilidade

#### Configurações Personalizadas

- Configure qualquer configuração M-de-N
- Adapte às suas necessidades de segurança específicas
- Estruturas de aprovação flexíveis

## Processo de Transação

### Iniciando uma Transação

1. **Criar Transação**: Rascunhe a transação no TBC Wallet
2. **Enviar para Aprovação**: A transação é enfileirada para aprovação multi-sig
3. **Notificação**: Todos os detentores de chaves recebem notificação
4. **Aprovação**: Os detentores de chaves revisam e aprovam/rejeitam
5. **Execução**: A transação é executada uma vez que o limiar é atingido

### Interface de Aprovação

- **Detalhes da Transação**: Visualize informações completas da transação
- **Aprovar/Rejeitar**: Interface de aprovação simples
- **Comentários**: Adicione notas ou comentários às aprovações
- **Histórico**: Rastreie todas as atividades de aprovação

## Perguntas Frequentes

### Quantas assinaturas eu preciso?

O número de assinaturas necessárias depende da sua configuração. Configurações comuns são 2-de-3 ou 3-de-5, mas você pode personalizar com base em suas necessidades.

### O que acontece se eu perder uma chave?

Se você perder uma chave mas ainda tiver chaves suficientes para atender ao limiar, pode continuar operando normalmente. No entanto, você deve substituir a chave perdida para manter a redundância.

### Posso mudar a configuração multi-sig?

Sim, você pode modificar sua configuração multi-sig, mas isso tipicamente requer aprovação dos detentores de chaves existentes para garantir a segurança.

### Multi-sig é mais lento que transações regulares?

Transações multi-sig podem levar um pouco mais de tempo, pois requerem múltiplas aprovações, mas a segurança aprimorada vale o atraso mínimo.

### Posso usar multi-sig com carteiras de hardware?

Sim, o TBC Wallet suporta integração de carteiras de hardware para configurações multi-assinatura, fornecendo ainda maior segurança.
